# 内容安全与 Actions 安全

## 外部贡献安全基线

1. 申请通过的投稿者可加入仓库协作者，以便直接分支 + PR 投稿。
2. 协作者仍不得直接推送 `main`，必须通过 PR 审核合并。
3. PR 工作流使用 `pull_request` 事件运行，避免不可信代码获取高权限。
4. Secrets 不应在外部 PR 的校验任务中暴露。

## 部署安全基线

1. 部署工作流仅在 `main` 分支 `push` 触发。
2. 部署权限仅保留 `pages:write`、`id-token:write`、`contents:read`。
3. 启用并发组，避免多个部署互相覆盖。

## 内容安全建议

1. 禁止提交凭据、令牌、内部密钥或敏感截图。
2. 对流程文档中的权限信息按“最小可见”原则编写。
3. 对安全策略改动，必须由责任人复核。
